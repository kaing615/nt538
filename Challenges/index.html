📁 Challenge 1: Tổng số chia hết cho 3

🎯 Mục tiêu

Đọc file, tìm các số nguyên dương chia hết cho 3, tính tổng. Xử lý song song bằng cách chia file theo byte range.

📋 Quy tắc lọc

┌─────────────────────────────────────────────┐ │ Số hợp lệ phải thỏa mãn TẤT CẢ điều kiện:│ ├─────────────────────────────────────────────┤ │ 1. Có thể chuyển sang float │ │ 2. Lớn hơn 0 │ │ 3. Là số nguyên (val.is_integer() == True)│ │ 4. Chia hết cho 3 │ └─────────────────────────────────────────────┘

📝 Thuật toán từng bước

Đọc dòng đầu
Dòng đầu tiên chứa số lượng phần tử n, bỏ qua.
File: n=7 1 2 3 6 9 12 15 18 21 ↑──┘ Dòng đầu (bỏ qua)
Chia file thành 16 đoạn byte
Tính chunk_size = data_size / 16
┌──────┬──────┬──────┬──────┬──────┬──────┐ │ P1 │ P2 │ P3 │ P4 │ ... │ P16 │ │0-10 │11-20│21-30│31-40│ │ │ └──────┴──────┴──────┴──────┴──────┴──────┘
Điều chỉnh điểm cắt
Di chuyển điểm cắt đến khoảng trắng gần nhất, không cắt giữa một số.
⚠️ Quan trọng: Nếu cắt ở giữa số "123", sẽ tách thành "12" và "3" → sai!
❌ Cắt sai: "1 2 3 |6 9" → 3|6 bị tách ✅ Cắt đúng: "1 2 |3 6 9" → 3 6 9 còn nguyên
Mỗi process xử lý đoạn của mình
Đọc chunk, split(), kiểm tra điều kiện, tính tổng cục bộ.
Process chính cộng kết quả
sum(results) = 3 + 27 + 54 = 84

💻 Code minh họa

def worker_task(args): file_path, start_byte, end_byte = args total = 0 with open(file_path, 'r') as f: f.seek(start_byte) chunk = f.read(end_byte - start_byte) for s in chunk.split(): try: val = float(s) if val > 0 and val.is_integer() and int(val) % 3 == 0: total += int(val) except ValueError: continue return total

🔢 Challenge 2: Fibonacci Modulo

🎯 Mục tiêu

Tính F(A_i) mod Q cho nhiều truy vấn, với F là dãy Fibonacci (F(0)=0, F(1)=1).

⚡ Fast Doubling Algorithm

Tính F(n) trong O(log n) bằng công thức:

F(2k) = F(k) × [2F(k+1) - F(k)]
F(2k+1) = F(k)² + F(k+1)²

📝 Ví dụ: Tính F(10) mod 100

Binary của 10
10 = 1010 (từ phải sang)
Bit: 1 0 1 0 Vị trí: 3 2 1 0 Bắt đầu: (F(0), F(1)) = (0, 1)
Bit 0 (n & 1 = 0)
k=1, dùng (F(0), F(1)) → (F(0), F(1)) vì bit=0
a = F(0) = 0, b = F(1) = 1 c = F(0) × (2×1 - 0) = 0 → F(0) d = 0² + 1² = 1 → F(1)
Bit 1 (n & 2 = 1)
k=2, bit=1 nên dùng (F(1), F(2))
a = F(1) = 1, b = F(2) = 1 c = F(2) = 1, d = F(3) = 2 Vì bit=1: return (d, c+d) = (2, 3)
Bit 2 (n & 4 = 0)
k=4, bit=0 nên dùng (F(2), F(3))
a = F(2) = 1, b = F(3) = 2 c = F(4) = 3, d = F(5) = 5 Vì bit=0: return (c, d) = (3, 5)
Bit 3 (n & 8 = 1)
k=8, bit=1 nên dùng (F(4), F(5))
a = F(4) = 3, b = F(5) = 5 c = F(8) = 21, d = F(9) = 34 Vì bit=1: return (d, c+d) = (34, 55)
✅ Kết quả
F(10) mod 100 = 55

🚀 Tối ưu hóa

Trường hợpPhương phápĐộ phức tạp
Q = 1Return [0]*nO(1)
Q = 2Period = 3O(n)
Q ≤ 80000Pisano PeriodO(q)
Range ≤ 300000Precompute liên tụcO(range)
LớnFast doubling + Gap tableO(log n)

📐 Challenge 3: Nhân ma trận song song

🎯 Mục tiêu

Nhân hai ma trận vuông A × B, trả về ma trận C cùng đường chéo và tổng.

📝 Công thức

C[i][j] = Σ A[i][k] × B[k][j]

💡 Tối ưu: Transpose ma trận B

Thay vì truy cập cột của B (không liên tục trong memory), ta chuyển vị B thành BT:

┌─────────────────┐ ┌─────────────────┐ │ B = [[5,6], │ │ BT = [[5,7], │ │ [7,8]] │ → │ [6,8]] │ └─────────────────┘ └─────────────────┘ C[i][j] = A[i] · BT[j] (dot product!)

📝 Ví dụ: A × B

Ma trận ban đầu
0 1 A = [0] 1 2 [1] 3 4 0 1 B = [0] 5 6 [1] 7 8 BT = [0] 5 7 [1] 6 8
Tính C[0][0]
C[0][0] = A[0] · BT[0]
= [1, 2] · [5, 7] = 1×5 + 2×7 = 5 + 14 = 19
Tính C[0][1]
C[0][1] = A[0] · BT[1]
= [1, 2] · [6, 8] = 1×6 + 2×8 = 6 + 16 = 22
Tính C[1][0]
C[1][0] = A[1] · BT[0]
= [3, 4] · [5, 7] = 3×5 + 4×7 = 15 + 28 = 43
Tính C[1][1]
C[1][1] = A[1] · BT[1]
= [3, 4] · [6, 8] = 3×6 + 4×8 = 18 + 32 = 50
✅ Kết quả
C = A × B = [[19, 22], [43, 50]] Đường chéo chính: [19, 50] Đường chéo phụ: [22, 43] Tổng: 134

🔄 Xử lý song song

┌─────────────────────────────────────────────┐ │ Process 1: Tính dòng 0 (C[0][0], C[0][1]) │ │ Process 2: Tính dòng 1 (C[1][0], C[1][1])│ │ ... │ │ Process N: Tính dòng N-1 │ └─────────────────────────────────────────────┘

📊 Challenge 4: Sắp xếp Record

🎯 Mục tiêu

Sắp xếp n record (v, label) theo thứ tự:

  1. v giảm dần
  2. label tăng dần
  3. Ổn định theo index ban đầu

🔑 Encode thành Sort Key

key = (-v, label, i, v, label)
Giải thích:
• -v: Python sort tăng dần → -v tăng dần = v giảm dần
• label: sort tăng dần theo bytes
• i: index gốc → độ ổn định
• v, label: để phục hồi kết quả

📝 Ví dụ

Input records
Index 0: (100, "A") Index 1: (50, "B") Index 2: (100, "A") Index 3: (75, "C")
Encode mỗi record
[0] (100, "A") → (-100, b"A", 0, 100, b"A") [1] (50, "B") → (-50, b"B", 1, 50, b"B") [2] (100, "A") → (-100, b"A", 2, 100, b"A") [3] (75, "C") → (-75, b"C", 3, 75, b"C")
Sort theo tuple
Python so sánh từng phần tử:
So sánh [0] và [2]: cùng (-100, b"A") → So sánh i: 0 < 2 → [0] đứng trước [2] Sort kết quả: 1. (-100, b"A", 0, ...) ← [0] 2. (-100, b"A", 2, ...) ← [2] 3. (-75, b"C", 3, ...) ← [3] 4. (-50, b"B", 1, ...) ← [1]
✅ Kết quả cuối cùng
(100, "A") ← v giảm dần: 100 (100, "A") ← cùng v, label A = B (so sánh bytes) (75, "C") ← v = 75 (50, "B") ← v giảm dần: 50

🔍 Challenge 5: Tìm vị trí đầu tiên

🎯 Mục tiêu

Tìm vị trí (index) của key trong dãy số. Trả về -1 nếu không tìm thấy.

⚠️ Vấn đề: Token Boundary

Nếu chỉ dùng str.find(key), sẽ nhầm "10" trong "110"!

┌─────────────────────────────────────────────┐ │ Body: "10 110 1000 100" │ │ │ │ Tìm "10": │ │ ❌ body.find("10") = 0 │ │ → Nhầm vì tìm thấy "10" trong "110" │ │ │ │ ✅ Đúng: index = 0 vì "10" là token riêng│ │ (có space trước và sau) │ └─────────────────────────────────────────────┘

📝 Thuật toán từng bước

Tìm vị trí bằng substring search
Dùng body.find(key) để tìm vị trí.
Kiểm tra boundary trước key
if pos > 0 and not body[pos-1].isspace(): # Sai! Đây là substring, không phải token # Ví dụ: "110" chứa "10" nhưng không phải token "10"
Kiểm tra boundary sau key
if pos + len(key) < len(body) and not body[pos + len(key)].isspace(): # Sai! Tiếp theo không phải space # Ví dụ: "100" bắt đầu bằng "10" nhưng không phải "10"
Đếm số token
Đếm số token trước vị trí tìm thấy để ra index.

📝 Ví dụ chi tiết

Tìm key="10" trong "10 110 1000 100"

Token 0: "10"
pos = 0, tìm thấy
Trước: đầu string ✓
Sau: space ✓

📍 Challenge 6: Closest Pair

🎯 Mục tiêu

Tìm khoảng cách Euclid nhỏ nhất giữa 2 điểm trong không gian 2D.

🔢 Encode điểm

encoded = (x + OFFSET) << 32 | (y + OFFSET)
OFFSET = 10⁹ để xử lý tọa độ âm
<< 32: dịch trái 32 bits cho x
|: OR với y

📝 Ví dụ: Encode và Sort

Điểm ban đầu
P1 = (1, 2) P2 = (3, 4) P3 = (5, 1) P4 = (2, 3) P5 = (4, 5)
Encode
P1: ((1+1e9) << 32) | (2+1e9) = 4294967301294967298 P2: ((3+1e9) << 32) | (4+1e9) = 4294967309884901892 P3: ((5+1e9) << 32) | (1+1e9) = 4294967318474836481 P4: ((2+1e9) << 32) | (3+1e9) = 4294967305589934595 P5: ((4+1e9) << 32) | (5+1e9) = 4294967314179869189
Sort (theo x, rồi theo y)
P1(1,2) → P4(2,3) → P2(3,4) → P5(4,5) → P3(5,1)
Decode thành arrays
xs = [1, 2, 3, 4, 5] ys = [2, 3, 4, 5, 1]

🔍 Scan tìm Closest Pair

Với mỗi điểm, chỉ so sánh với 8 điểm kế tiếp (heuristic):

i=0, điểm (1,2)
So sánh với (2,3): dist = √((2-1)² + (3-2)²) = √2 ≈ 1.414
So sánh với (3,4): dist = √((3-1)² + (4-2)²) = √8 ≈ 2.828
Best = 1.414
i=1, điểm (2,3)
So sánh với (3,4): √2 ≈ 1.414 = Best
So sánh với (4,5): √8 ≈ 2.828
Best = 1.414 (không đổi)
i=2, điểm (3,4)
So sánh với (4,5): √2 ≈ 1.414 = Best
So sánh với (5,1): √25 = 5
Best = 1.414
✅ Kết quả
Closest pair: (1,2) và (2,3)
Khoảng cách: √2 ≈ 1.4142
Y │ 5 · P5(4,5) │ 4 · P2(3,4) │ 3 · P4(2,3) │ 2 · P1(1,2)──·── P4(2,3) distance = √2 │ 1 · P3(5,1) └──┴──┴──┴──┴── X 1 2 3 4 5

🔄 Transpose khi cần

Nếu có nhiều điểm cùng hoành độ (max_x_run > 8), đảo vai trò x và y rồi scan lại:

┌─────────────────────────────────────────────┐ │ swapped[i] = (ys[i] << 32) | xs[i] │ │ │ │ Sort → điểm xếp theo y, rồi theo x │ │ Scan lại với 8 láng giềng │ │ Lấy min(Best, Transposed_Best) │ └─────────────────────────────────────────────┘
