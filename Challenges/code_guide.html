🔢 Challenge 2: Fast Doubling Fibonacci

Mục đích
Tính F(n) mod q nhanh O(log n)
Công thức
F(2k), F(2k+1)
Tối ưu
Pisano, Gap Table

fib_pair_mod - Fast Doubling

challenge2.py
17def fib_pair_mod(n, q):
18 a = 0
19 b = 1
20
21 if n == 0:
22 return 0, 1
23
24 mask = 1 << (n.bit_length() - 1)
25
26 while mask:
27 t = b + b - a
28 c = (a * t) % q # F(2k)
29 d = (a * a + b * b) % q # F(2k+1)
30
31 if n & mask:
32 a, b = d, (c + d) % q
33 else:
34 a, b = c, d
35
36 mask >>= 1
37
38 return a, b
📍 Dòng 17-19: Khởi tạo
a = 0 = F(0)
b = 1 = F(1)

Hàm trả về cặp (F(n), F(n+1)) mod q
📍 Dòng 21-22: Base case
Nếu n = 0: return (0, 1)

F(0) = 0, F(1) = 1
📍 Dòng 24: Lấy bit cao nhất
mask = 1 << (n.bit_length() - 1)

Ví dụ: n = 10 (1010)
bit_length = 4
mask = 1 << 3 = 8 (1000)

Mask dùng để duyệt từng bit của n, từ MSB đến LSB.
📍 Dòng 27-29: Fast doubling
Tính cặp (F(2k), F(2k+1)):

c = a * (2*b - a) % q → F(2k)
d = a² + b² % q → F(2k+1)

Đây là công thức toán học của fast doubling.
📍 Dòng 31-34: Xử lý bit
if n & mask: - Bit hiện tại = 1?

• Bit = 1: a, b = d, c+d → dùng F(2k+1), F(2k+2)
• Bit = 0: a, b = c, d → dùng F(2k), F(2k+1)

mask >>= 1 - Di chuyển sang bit kế tiếp
📍 Dòng 38: Kết quả
Trả về (a, b) = (F(n), F(n+1)) mod q

📍 Challenge 6: Closest Pair

_encode_dataset - Mã hóa điểm

challenge6.py
299def _encode_dataset(data, left, right):
300 offset = 1000000000
301 return [
302 ((data[i] + offset) << 32) | (data[i + 1 + offset)
303 for i in range(left, right, 2)
304 ]
📍 Dòng 299-304: Encode điểm
encoded = ((x + OFFSET) << 32) | (y + OFFSET)

Ví dụ: P(5, 3)
• x + OFFSET = 5 + 10⁹ = 1000000005
• y + OFFSET = 3 + 10⁹ = 1000000003
• x << 32 = 1000000005 << 32
• encoded = (x << 32) | y

🔑 Tại sao dùng 64-bit?
• 32 bits cho x: cho phép tọa độ ±2³¹
• 32 bits cho y: cho phép tọa độ ±2³¹
• Sort encoded = sort theo x, rồi theo y
